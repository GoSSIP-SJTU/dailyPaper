今天带大家去阅读一篇来自AsiaCCS 2020会议上关于代码反编译的研究论文A Comb for Decompiled C Code



本文作者来自意大利，包括了rev.ng（https://rev.ng/）这个反编译器开发项目的主要参与人员，作为Hex-rays的挑战者，作者对自己的研究工作也进行了宣传，这篇论文主要介绍了rev.ng反编译器如何实现高质量的反编译代码（包括goto-free、人工容易理解等特性），作者主要通过名为control flow combing的技术来生成更为简洁的反编译代码


rev.ng反编译的核心思想在于首先强制要求反编译依赖的CFG满足一定的属性，在满足这些属性的情况下，得到的反汇编代码自然就会清晰。而不满足特定属性，会导致代码中出现难以理解的结构，例如下图中node 2是一个条件分支，那么node 3显然应该是从node 2的条件分支中的一个分支达到，然而图中node 1也可以到达node 3，这隐含表明了在高级结构中必须引入一个goto才能实现，因而破坏了代码的可读性。rev.ng反编译为了杜绝这种情况，就规定所使用的CFG必须满足相关属性（在这里是定义了一种称为Diamond Shape的属性 ，也是control flow combing要求的核心属性）




还有一种情况是针对return的特殊处理，下图展示了一个反编译的问题，在1-5行的双重条件判断分支里面包含了一个return，在生成CFG时，如果把它当做普通的情况处理，就会生成（b）图里面这种将complex code snippet（node C）复制两份的结果，造成了反编译结果的急剧膨胀。而rev.ng反编译则针对这个情况进行了专门处理，只复制node R，这样就避免了代码膨胀的情况发生


作者在评估部分对标了Hex-rays和Ghidra，主要鄙视了竞争对手的反编译结果中海量的goto


感兴趣的童鞋们可以去https://github.com/revng/revng下载代码来试用
