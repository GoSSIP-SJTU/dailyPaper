原创 G6 上海交大软件安全研究组GoSSIP
昨天

今天介绍的是一篇关于如何对(Re-)randomization的安全性及间隔时间进行量化分析的一篇文章。


linux中的ASLR属于Coarse-grained ASLR，其只会将module/segment作为整体进行随机化，然而攻击者只需要泄露出其中的一个地址，即可在ROP时利用整个模块中的gadget。


因此，安全研究人员设计出了多种Fine-grained ASLR，即以functions、basic blocks、instructions、machine register为粒度进行随机化，以缓解ROP类攻击。再后来，又出现了re-randomization的方案，即每间隔一定时间就重新进行随机化，以减小攻击者的时间窗口。这篇文章提出了一种量化上述随机化方案的安全性及re-randomization中间隔时间的方法。


作者为gadget进行了分类，定义了图灵完备gadget集合：

  • re-randomization的间隔时间上界，即攻击者无法在上界时间内获得Turing-complete, priority, MOV TC, or payload gadget set

  • gadget的副作用，gadget中出核心指令外，可能会包含破坏其他寄存器/内存状态指令

可以看到，攻击的流程主要分为三个部分：memory layout derandomization, system access, and payload generation。其中在Memory Layout Derandomization阶段：

  1.  Basic ROP：由于Coarse-grained ASLR只能以module/segment的粒度进行随机化，因此攻击者只需要leak出module内部的一个地址即可计算出module的基地址，从而根据预先设置的偏移找到gadgets。

  2. Just-in-time ROP：Fine-grained ASLR会对code page进行随机化，因此不可能通过线性扫描的方式去探索code page（因为可能读到unmapped memory）。研究人员在2013年JIT-ROP中提出了一种递归下降的code page探索方法，攻击者可以控制一个指向code page的指针，通过这个指针，不断的去检测call jmp指令的跳转地址，从而发现新的code page。如下图所示：

测量的方式包括：

  • Methodology for Derandomization

  • Gadget selection：gadget的分类


  • Methodology for single-round randomization experiments：单轮fine-grained code randomization的方案中，统计上述各种gadget出现的数量。

  • Methodology for re-randomization experiments：选取100个re-randomized后连续的地址空间snapshot进行手动分析。

  • Methodology for System Access：作者通过统计可用的system gadget和在堆栈、data段中vulnerable library pointers的数量，来量化访问privileged operations的难度。

  • Methodology for Payload Generation：作者通过测量单个gadget的质量以评估生存gadget chain的质量。作者通过register corruption analysis的方式来评价单个gadget的质量。


作者从Re-randomization间隔时间上界、指针泄漏位置的影响、对可用gadget的影响、运行开销的影响、gadget chain质量的影响和获得libc指针的难易程度这六个方面，对应用在多个应用程序、库上的数种随机化方案进行评估。
