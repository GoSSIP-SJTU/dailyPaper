G.O.S.S.I.P 学术论文推荐 2020-07-28

RomanGol 上海交大软件安全研究组GoSSIP
6天前

今天的学术论文推荐，带大家走进微软研究院，看看他们发表在2020年ATC会议上的研究论文


本文关注的是如何在已部署的系统上分析内核的错误，通常情况下，分析人员只有memory dump等有限的信息，如何通过这些信息推测出错误发生的原因呢？作者基于硬件追踪辅助，在Windows操作系统上实现了一套名为Kernel REPT的控制流追踪系统，通过记录执行的路径，再加上软件模拟来恢复数据流信息，最终实现了反向调试（reverse debugging），支持了高效的内核错误根本原因发现。


值得一提的是，本文的基础是第一作者和第三作者在2018年拿到OSDI最佳论文的工作REPT的延续（https://www.usenix.org/system/files/osdi18-cui.pdf，相关论文笔记：https://hoblovski.github.io/2019/01/05/REPT-Reverse-Debugging-of-Failures-in-Deployed-Software.html）。REPT的基本思想也延续到了本文中（即硬件辅助来做 control flow tracing）。

最后，来自微软的研究论文的福利便是：

We have deployed Kernel REPT in the ecosystem of Microsoft Windows. 

  • On Windows, we released the kernel driver that conﬁgures Intel PT tracing and the ETW context switch logger for the kernel, and a user-mode daemon that communicates with WER to decide when to start/stop the driver. These components were released as part of Windows 10 version 1803

  • On the WER service, we added support for requesting Intel PT traces for a given kernel failure.

  • On Windows Debugger, we implemented Kernel REPT’s interactive reverse debugging by extending REPT’s debug-
