今天要给大家推荐2014年由奥斯卡影帝阿德里安布罗迪主演，讲述传奇魔术师、逃脱大师Houdini的两集迷你剧……打住打住，今天要给大家推荐的是2019年CCS上一篇关于如何逃逸Linux Control Groups管控机制的研究论文



Linux Control Groups (cgroup) 是 container 技术中非常重要的机制，用于限制每个 container 的资源使用。cgroup 将进程分组，再用 controller 管理每个分组相应的系统资源，例如 CPU 和内存的使用。换言之，cgroup 可以计算每个 container 的资源使用量，并限制每个 container 可以使用的最大资源量。cgroup 监管资源的使用。cgroup 防止 container 大量消耗 host 上的系统资源，避免一些 Denial-of-Service 攻击。


然而，论文作者作者分别利用了 kernel 的异常处理机制、数据同步机制、logging 系统服务、docker 引擎以及 softirqs 中断处理机制，给系统引入了一些复杂的工作量，消耗主机大量资源（在这过程中，并没有使得攻击者拥有更多的可用资源，因此只能算作一种“损人不利己”的攻击）。作者在本地和 Amazon EC2 cloud 上都进行了实验，发现即使在多个 cgroup controller 的控制之下，恶意的 container 依然可以大量地耗尽主机的资源，消耗 CPU，产生巨大的 I/O，并且这些行为都不会被计算在 container 的资源使用量上。



作者提出四种 exploiting strategies，用以摆脱 cgroup 的限制，其insight在于将 workload 运行在kernel thread、系统服务、异常处理等不受cgroup controller限制的地方（尽管可能具体的workload是由container里面的代码发起的），这些特定的任务执行方式，就会造成大量的资源消耗而得不到限制。在实践中，作者在 docker container 中基于上面四条策略设计出了五个可行的 case，对具体的本地和云端服务器发起了攻击，消耗了大量资源：
