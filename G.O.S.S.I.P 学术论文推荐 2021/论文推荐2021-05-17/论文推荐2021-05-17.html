今天给大家推荐的是来自USENIX Security 2021的一篇关于 Node.js 安全分析的文章——"Abusing Hidden Properties to Attack the Node.js Ecosystem"。Node.js 因为其动态语言和高性能 JavaScript 运行时等特性被广泛应用在桌面环境和服务端环境，但目前缺乏针对 JavaScript 和 Node.js 的相关安全隐患的研究。本文作者对 Node.js 进行了系统性的安全分析，设计并发明了一种新的 Node.js 中的攻击——隐藏属性滥用 (Hidden Property Abuse, HPA)，此外作者还开发了一个针对 Node.js 的漏洞发现与自动化利用生成的工具——LYNX，以帮助开发者防御 HPA 攻击。


Node.js 应用程序主要使用 JavaScript 这种动态语言编写，近年针对动态语言例如 PHP、Ruby 的对象安全问题 CWE-915，即内部对象属性被外部用户输入所修改的问题研究较多，但是却没有针对 JavaScript 的研究。本文中，作者确认了 CWE-915 也同样存在于 JavaScript 与 Node.js 中。为了证明这种安全问题，作者引入了一种新的攻击手段——HPA；主要原理是由于Node.js 应用应用实现了对象分享（例如数据反序列化），远程 Web 攻击者可以通过合法的接口发送精心构造的数据达到攻击效果。

  • 受限于应用的对象属性操控：Web 应用错误地将特定的对象属性暴露使得攻击者可控，攻击者可能通过这种攻击向量来将特定的对象属性传递到内部对象，从而访问敏感的接口。

  • 原型链劫持：攻击者劫持特定对象的原型链，加入受攻击者控制的特定属性


作者开发了一个针对Node.js的漏洞发现与利用工具——LYNX，LYNX 结合了动态分析与静态分析的优点，可以自动化分析应用并针对漏洞生成漏洞利用，如图3所示。

（1）如何发现 Node.js 应用中的隐藏属性：使用动态分析的方法标记用户输入的对象，然后使用静态分析消除漏报。

（2）如何在大量隐藏属性中找到有价值或可以利用的属性：利用符号执行来探索所有执行路径，检查敏感行为，最后生成漏洞利用。

（3）漏洞利用：

  • 漏洞利用模板生成：作者直接利用原始对象作为模版，并在对象中插入属性。

  • 探索攻击路径：作者总结了六种敏感函数触发点（Sink）：1）敏感数据库查询方法；2）敏感文件系统操作方法；3）代码执行接口；4）模块引用结果；5）全局方法或变量；6）循环条件。作者使用搜集好的经过分类的接口进行识别，覆盖了返回值操控、全局变量修改、循环条件操控三种恶意行为。


作者使用了 102 个真实的 Node.js 应用评估了 LYNX，最后找到了 15 个先前未知的漏洞，获得了 10 个分配的 CVE 编号，其中有 8 个被评级为高危或严重，7 个已经被程序维护者修复。
